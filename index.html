Welcome to the Boulder Project, an Intent-based NBI Project

Questions About Boulder

What is Boulder? Boulder is an ONF sponsored project under the OpenSourceSDN umbrella for Intent-based Northbounds of SDN controllers. It focuses on semantics and information models to enable applications to tell networks what to do and not how to do it.

What is it designed to do? Boulder was designed to provide Intent portability across different controller environments. It provides a layer which shields application developers from any API changes or northbound evolution. Moreover it provides both a declarative or imperative approach for writing applications through its scripting environment.

Does Boulder solve an issue faced by the industry? If so, what is the issue and how will Boulder help? Boulder is trying to solve application portability, as well as SDN application co-existence. In a similar fashion that virtual machine images are portable compute units, boulder will aim at providing a seamless experience across multiple execution environments when working with SDN controllers.

Does Boulder leverage ODL or any other SDN organizationâ€™s work? If so, who? Boulder works closely with the OpenDaylight NIC projects which provides a native implementation of the Information Model and therefore a nicer integration and a set if richer features.

Where any companies deeply involved in the development of Boulder? It has been mainly Inocybe and HP so far. Chriz Janz of Ciena is now chair of the information model within ONF (Intent NBI) and will likely get involved later. However for the OpenSourceSDN project itself it is Inocybe with HP. So the exhaustive list would be Inocybe Technologies, HP, Ciena (Dave L? your input here on the politics is more than welcome) might be hard to get approval.

Are there any licensing details we should include? Boulder is dual-licensed under both Apache Software License and Eclipse Public License so that it can be leveraged in other controller projects.

Boulder was demoed at the ODL Summit. Was there any feedback following the demo that we should reference in the release? Boulder was demoed both at ONS and ODL Summit. However it is not yet officially released (hoping to release it around SDN World Congress time frame) . So people have expressed enough interest to pursue this a little further. Moreover, yea many application developers came to ask when it would be ready for them to leverage this for their applications.
